# Album Clip Timeline Builder v1.0.1 User Guide

Album Clip Timeline Builder は、複数の音声クリップ（WAV / MP3）をアルバム形式のタイムラインへ配置し、実音開始・実音終了を基準に曲間を整え、`album.wav`、`album_subtitles.srt`、`timesheet.txt`、`tracklist.txt`、`project.json`、`offset_image.txt` を出力するためのローカルHTMLアプリです。

This tool is a local browser-based album timeline builder. It is designed for music-production and video-production workflows where several generated or edited tracks need to be assembled into one album-length audio file, with subtitle and timing helper files exported from the same timeline.

## このツールでできること

Album Clip Timeline Builder では、次の作業ができます。

- 複数のWAV / MP3ファイルを選択する
- ファイル名昇順で曲リストを作成する
- 曲順を手動で入れ替える
- 音声ファイルを読み込み、各曲の実音開始・実音終了を検出する
- 前曲の実音終了から次曲の実音開始までが指定秒数になるように自動配置する
- アルバム全体の簡易波形を確認する
- 曲間の拡大波形を確認する
- 最終曲末尾の波形を `END` 区画から確認する
- 実音開始・実音終了を、10ms、100ms、または波形上の約1px単位で微調整する
- 選択曲以降を指定曲間で再配置する
- 書き出し範囲をプレビュー再生する
- 1本の `album.wav` を出力する
- 複数のSRTファイルを、現在のタイムラインに合わせて1本の `album_subtitles.srt` へ合成する
- 各音源の00:00位置を `offset_image.txt` として出力する
- 画像差し替え用の `timesheet.txt` を出力する
- YouTube説明欄などに使う `tracklist.txt` を出力する
- 作業状態を `project.json` として保存する
- `project.json` を読み込み、同じ作業状態を再現する
- `project.json` 読み込み時に必要な音声ファイル・SRTファイル一覧を確認する
- 日本語 / English のUIを切り替える

## このツールが向いている用途

このツールは、次のような用途に向いています。

- AI生成曲など、複数の単曲音源を1つのアルバム音源へまとめる
- 曲ごとに冒頭・末尾の無音長がばらついている音源を、実音基準で自然な曲間へ整える
- アルバム動画、長尺YouTube動画、ライブ配信用BGMなどで、複数曲を1本の音声にまとめる
- 曲ごとのSRT字幕を、アルバム全体のタイムラインへ合わせて結合する
- 動画編集ソフトで曲ごとに画像を差し替えるためのタイムシートを作る
- 一部の音源だけ差し替えながら、同じ曲順・曲間・SRT対応を再利用する

このツールはDAWではありません。フェード、クロスフェード、音量補正、ミキシング、マスタリングを目的としたものではありません。目的は、複数の音声クリップを「どこに置くか」「実音区間をどこからどこまでとみなすか」を整理し、そこから音声・字幕・タイミング補助ファイルを一括して出力することです。

## 重要な設計思想

このアプリの中心にある考え方は、単なるファイル連結ではなく、**実音基準のタイムライン配置**です。

通常の単純連結では、ある曲の末尾無音と次曲の冒頭無音がそのまま残ります。AI生成音源や変換後音源では、曲によって冒頭・末尾の無音長が不規則なことがあります。その状態で単純連結すると、曲間が不自然に長くなったり、逆に短すぎたりします。

Album Clip Timeline Builder では、各曲について次の3つを分けて扱います。

- 音源ファイル全体の範囲
- 実音開始
- 実音終了

そして、前曲の実音終了から次曲の実音開始までが、指定した標準曲間になるように各音源をタイムラインへ配置します。

たとえば、標準曲間を2.7秒にした場合、前曲のファイル末尾から次曲ファイル先頭までが2.7秒になるのではありません。前曲の「実音終了」と次曲の「実音開始」の間が2.7秒になるように配置されます。

このため、タイムライン上では前曲のファイル末尾無音と次曲のファイル先頭無音が重なることがあります。しかし、書き出し時には実音開始より前と実音終了より後をゼロサンプル無音として扱うため、不要な元音源の無音やクリックノイズを含めずに、整理されたアルバムWAVを作れます。

## プライバシーとローカル処理

このツールはローカルHTMLアプリです。`index.html` をブラウザで開き、ユーザーが選択した音声ファイルやSRTファイルをブラウザ内で処理します。

このアプリ自体は、音声ファイル、字幕ファイル、生成したWAV、project.jsonをサーバーへアップロードしません。GitHub Pagesで公開した場合でも、配信されるのはHTMLとドキュメントだけです。ユーザーが選択したローカルファイルは、ユーザーのブラウザ内で処理されます。

ただし、ブラウザのセキュリティ仕様により、project.jsonに記録されたファイル名やパスから元ファイルを自動で再取得することはできません。プロジェクトを再開するときは、ユーザーが音声ファイルやSRTファイルを再選択する必要があります。

## 推奨環境

推奨環境は、Windows上の最新の Google Chrome または Microsoft Edge です。

このツールは、File API、Web Audio API、Canvas、Blobダウンロードなどのブラウザ機能に依存します。WAV入力の一部はアプリ内パーサーで処理します。MP3入力はブラウザの音声デコード機能に依存します。

FirefoxやSafariでも一部機能は動作する可能性がありますが、リリース想定はChrome / Edgeです。特にMP3デコード結果やAudioContextのサンプルレートはブラウザ実装の影響を受けることがあります。

## 処理モード

Album Clip Timeline Builder には、内部的に2つの処理モードがあります。ユーザーが明示的に選ぶ必要はなく、入力ファイルの種類と形式に応じて自動判定されます。

### 標準WAVモード

全ファイルがWAVで、かつサンプルレート、チャンネル数、ビット深度、WAV形式が完全に一致している場合は、標準WAVモードになります。

このモードでは、できるだけ無加工に近い形で、元WAVの必要範囲を切り貼りします。音声サンプル自体のゲイン処理、フェード、クロスフェード、サンプルレート変換は行いません。

標準WAVモードは、次のような場合に向いています。

- すべて同じ条件で書き出したWAVをまとめる
- 音声変換を避けたい
- 出力WAVの形式を元音源に合わせたい
- できるだけシンプルで予測可能な書き出しをしたい

### デコードモード

MP3が含まれる場合、またはWAV同士の形式が一致しない場合は、デコードモードになります。

このモードでは、すべての入力音声を内部的にFloat音声として扱い、必要に応じてサンプルレートとチャンネル数を揃えてからWAVとして書き出します。

デコードモードの出力形式は、入力ファイル群から次の方針で決まります。

- サンプルレートは、デコード後の入力群の最大値に合わせます
- チャンネル数は、デコード後の入力群の最大値に合わせます
- WAV入力が含まれる場合、WAV入力群の最上位ビット深度を反映します
- 32-bit float WAVが含まれる場合は、32-bit float WAV出力になります
- MP3のみの場合は、16-bit PCM WAV出力になります

MP3にはWAVのようなPCMビット深度がありません。そのため、MP3だけを読み込んだ場合は、実用上扱いやすい16-bit PCM WAVとして出力します。

### デコードモードの注意点

MP3は圧縮音声です。WAVのようにPCMデータをそのまま切り出してコピーすることはできません。必ずブラウザでデコードしてからWAVへ書き出します。

また、MP3にはエンコーダ遅延やパディングの問題があります。元WAVからMP3化した音源の場合、元WAVとMP3で完全に同じサンプル位置になるとは限りません。このアプリは無音検出と手動調整を備えているため実用上は吸収できますが、SRTや映像と極端に厳密な同期が必要な場合は、元WAVを使う方が安全です。

## 出力ファイル

### album.wav

現在のタイムラインに基づいて作成されるアルバムWAVです。

書き出し時は、各曲の実音開始より前と実音終了より後をゼロサンプルとして扱います。これは1曲目の先頭、最終曲の末尾にも同じように適用されます。

この設計により、元音源の冒頭無音・末尾無音・末尾クリックノイズなどを、実音範囲外として除外できます。

### album_subtitles.srt

選択した複数のSRTファイルを、現在の音声タイムラインに合わせて1本に合成したSRTです。

各SRTは、曲リストの順番とSRTリストの順番を1対1で対応させます。各字幕タイムスタンプには、その曲の「音源 00:00位置」が加算されます。

### offset_image.txt

各音源の00:00位置が、アルバムWAV上のどの時刻に対応するかを出力するテキストです。

形式は次の通りです。

```text
01	00:00,000
02	03:45,120
03	07:18,640
```

これは主に、外部のSRT結合バッチや他ツールで、曲別ファイルをアルバム全体の時刻へ変換するための基準として使えます。

### timesheet.txt

動画編集ソフトで、曲ごとの画像やタイトルカードを差し替えるためのタイムシートです。

形式は次の通りです。

```text
01	00:00,000	雨あがりの合図
02	03:48,250	次の曲名
03	07:21,430	さらに次の曲名
```

1列目はID、2列目は切替時刻、3列目は音源ファイル名から拡張子を除いた仮タイトルです。

1曲目は常に `00:00,000` です。2曲目以降は、前曲の実音終了位置と当該曲の実音開始位置の中央を出力します。つまり、曲間の中央で画像を切り替えるための目安になります。

### tracklist.txt

YouTubeの説明欄、note記事、配信告知文などに貼るための曲リスト元テキストです。

形式は次の通りです。

```text
【Track List】
00:00 雨あがりの合図
03:48 次の曲名
07:21 さらに次の曲名
```

時刻は `timesheet.txt` と同じ切替時刻を元にします。ただし、`tracklist.txt` では `MM:SS` 形式に丸めます。丸めは3捨4入です。399msまでは切り捨て、400ms以上は1秒切り上げます。

タイトル部分には、`timesheet.txt` と同じく、音源ファイル名から拡張子を除いた文字列が入ります。正式な曲名に整えたい場合は、書き出し後のテキストを編集してください。

### project.json

現在の作業状態を保存するJSONファイルです。

保存される主な情報は次の通りです。

- アプリ名
- バージョン
- 作成日時
- 処理モード
- 出力フォーマット
- 標準曲間
- 無音判定閾値
- 検出ブロック
- 曲順
- 音源ファイル名
- 実音開始・実音終了
- 音源 00:00位置
- SRTファイル対応情報

project.jsonには、音声ファイル本体やSRTファイル本体は含まれません。再読み込み時には、対応する音声ファイルやSRTファイルを別途選択してください。

## 基本ワークフロー

### 新規にアルバムWAVを作る

1. `index.html` をブラウザで開きます。
2. `ファイル選択` から、まとめたいWAV / MP3ファイルを選択します。
3. 曲リストにファイル名昇順で並びます。
4. 必要に応じて `上へ` / `下へ` で曲順を調整します。
5. `無音判定閾値` と `検出ブロック` を必要に応じて設定します。
6. `標準曲間` を設定します。
7. `音声読み込み・自動配置` を押します。
8. アルバム全体波形と曲リストを確認します。
9. 必要に応じて曲間編集で実音開始・実音終了を調整します。
10. `album.wav を出力` でアルバムWAVを書き出します。

### 字幕も合成する

1. 音声ファイルを読み込み、タイムラインを整えます。
2. `SRTファイル選択` から、曲ごとのSRTファイルを選択します。
3. SRTリストが曲リストと同じ順番になるように調整します。
4. `album_subtitles.srt を出力` を押します。

### 動画編集用のタイムシートを作る

1. 音声タイムラインを整えます。
2. `timesheet.txt を出力` を押します。
3. 動画編集ソフト上で、各行の時刻を画像・タイトルカードの切替目安として使います。

### YouTube説明欄用の曲リストを作る

1. 音声タイムラインを整えます。
2. `tracklist.txt を出力` を押します。
3. 出力されたテキストを、YouTubeの説明欄、note記事、告知文などに貼り付けます。
4. 必要に応じて、ファイル名由来の仮タイトルを正式タイトルへ編集します。

### 作業状態を保存する

1. 音声タイムライン、SRT対応、無音判定位置を整えます。
2. `project.json を出力` を押します。
3. 後で再編集する場合、このJSONを読み込みます。

### project.jsonから再開する

方法は2通りあります。

#### 先にproject.jsonを読む

1. `project.json を読み込み` を押します。
2. 必要な音声ファイル・SRTファイル一覧が表示されます。
3. 一覧を見ながら、対応する音声ファイルを選択します。
4. `音声読み込み・自動配置` を押します。
5. project.jsonの曲順・実音開始・実音終了・配置情報が適用されます。

#### 先に音声ファイルを読む

1. 対応する音声ファイルを選択します。
2. `音声読み込み・自動配置` を押します。
3. `project.json を読み込み` を押します。
4. 現在の音声リストに対してproject.jsonの設定が適用されます。

ファイル名が一致する場合は、project.jsonの曲順に合わせて並べ替えます。ファイル名が一致しないが曲数が同じ場合は、現在の曲順に対して設定を適用するか確認します。これは「一部の音源を差し替えて、同じ作業をやり直す」用途を想定した動作です。

## 読み込み・配置

### ファイル選択

WAV / MP3ファイルを複数選択します。選択直後は、ファイル名昇順で曲リストに並びます。

この段階では、まだ音声データ本体は完全には読み込まれていません。曲順の調整、削除、全クリアは可能です。

### 音声読み込み・自動配置

選択済みファイルを読み込み、無音検出と指定曲間配置までまとめて実行します。

この処理では、入力に応じて標準WAVモードまたはデコードモードが自動的に選ばれます。読み込み中は進捗バーとステータスメッセージが表示されます。

### 無音判定閾値

この値以下の音を無音として扱います。既定値は `-49 dBFS` です。

値を高くすると、小さめの音も無音扱いされやすくなります。たとえば、末尾のリバーブやノイズを切り落としたい場合は、少し高めにすることがあります。

値を低くすると、微小な残響やノイズも実音として残りやすくなります。余韻を残したい場合は低めにする方が安全です。

### 検出ブロック

無音検出でまとめて判定する時間単位です。既定値は `10 ms` です。

小さい値にすると細かく検出できますが、処理は重くなります。大きい値にすると検出が粗くなりますが、処理は軽くなります。音楽用途では、通常は10ms前後で十分です。

### 無音検出

現在読み込まれている音声について、実音開始と実音終了を再検出します。閾値や検出ブロックを変更した後に使います。

### 無音リセット

全曲の実音開始を0、実音終了を曲の長さに戻します。これは、実質的に単純結合に近い状態へ戻すための機能です。

### 標準曲間

前曲の実音終了から次曲の実音開始までの目標秒数です。

たとえば `2.700` 秒にすると、実音同士の間隔が2.7秒になるようにタイムライン配置されます。

### 指定曲間で配置

現在の実音開始・実音終了を使って、標準曲間になるようにタイムラインだけを再配置します。

無音検出をやり直すボタンではありません。手動で実音開始・終了を調整した後、全体の曲間をもう一度整えるときに使います。

## 無音検出と実音区間

このアプリでは、曲全体をそのまま書き出し範囲にするのではなく、実音開始・実音終了を検出します。

実音開始は、音が閾値を超えた最初の位置です。実音終了は、最後に音が閾値を超えた位置の後ろ側です。

ただし、自動検出は万能ではありません。次のような場合は、手動調整が必要になることがあります。

- 曲頭に非常に小さいノイズがある
- 曲末尾に長いリバーブがある
- 曲末尾にクリックノイズがある
- フェードアウトが長く続く
- AI生成音源の末尾に不要な残響や破綻音がある

拡大波形とプレビュー再生を使い、実際にどこからどこまでを実音として扱うべきか確認してください。

---

## タイムライン配置の考え方

各曲には、アルバム上の「音源 00:00位置」があります。これは、その音源ファイルの00:00が、アルバムWAV上のどの時刻に対応するかを表します。

この位置は、SRT合成や `offset_image.txt` の基準にもなります。

重要なのは、音源00:00位置と実際に聴こえ始める位置は同じとは限らないことです。音源冒頭に無音がある場合、実音開始は音源00:00より後ろになります。

タイムライン上の実音開始位置は、次の関係で決まります。

```text
アルバム上の実音開始 = 音源 00:00位置 + 音源内の実音開始
```

同様に、実音終了位置は次のようになります。

```text
アルバム上の実音終了 = 音源 00:00位置 + 音源内の実音終了
```

自動配置では、この「アルバム上の実音終了」と「次曲のアルバム上の実音開始」の間が、標準曲間になるように各曲の音源00:00位置を決めます。

---

## 曲リスト

曲リストには、現在の曲順とタイムライン情報が表示されます。

主な列は次の通りです。

- ファイル
- 長さ
- 実音開始
- 実音終了
- 音源 00:00位置
- 実音ギャップ

曲リストは、主に確認用です。実音開始・実音終了の細かな調整は、右ペインの曲間編集で行います。

曲を選択すると、右ペインの拡大波形がその曲の直前曲間を中心に表示されます。最終曲末尾を調整したい場合は、アルバム全体波形の右端にある `END` 区画を選択します。

---

## 波形・曲間プレビュー

### アルバム全体の概形

アルバム全体の簡易波形を表示します。各曲の書き出し範囲が枠で重ねて表示されます。

曲区画をクリックすると、その曲を選択し、下の拡大波形に曲間が表示されます。右端の `END` 区画をクリックすると、最終曲末尾の調整ビューに切り替わります。

### 選択曲の入り付近

選択曲の直前にある曲間を中心に、前後5秒を加えた高解像度波形を表示します。

ここでは、次のような情報を確認できます。

- 前クリップ範囲
- 後クリップ範囲
- 前クリップ実音終了
- 後クリップ実音開始
- 前クリップ末尾無音
- クリップ間ギャップ
- 後クリップ先頭無音
- dB目盛り

波形は、弱い信号も見やすいようにdB寄りの表示になっています。リバーブや小さな残響がどこまで残っているか確認しやすくするためです。

### 最終曲末尾

アルバム全体波形の右端 `END` 区画を選ぶと、最終曲末尾のビューになります。

ここでは、最終曲の実音終了とアルバム終端を確認できます。最終曲末尾のクリックノイズや不要な余韻を切りたい場合に使います。

### プレビュー再生

`範囲をプレビュー再生` を押すと、拡大波形に表示されている範囲を再生します。

プレビューは、書き出し結果に近い形で再生されます。つまり、実音範囲外はゼロサンプルとして扱われます。視覚的な境界だけでなく、実際に曲間が自然かどうかを耳で確認できます。

---

## 曲間編集

曲間編集は、右ペインの拡大波形下に表示されます。

通常の曲選択時には、次の2つを調整します。

- 前曲の実音終了
- 選択曲の実音開始

1曲目を選択している場合は、1曲目の実音開始を調整します。

`END` 区画を選択している場合は、最終曲の実音終了を調整します。

### 移動ボタン

- `←1px`
- `1px→`
- `-100ms`
- `-10ms`
- `+10ms`
- `+100ms`

`1px` は、現在の拡大波形表示における約1ピクセル分の時間量です。拡大範囲によって実際の時間幅は変わります。

### この曲間以降を指定曲間で再配置

曲間編集後に、選択中の曲間から後ろだけを標準曲間で再配置します。

たとえば、3曲目の入りを手動で調整した後、3曲目以降の曲間をもう一度整えたい場合に使います。全体を再配置する `指定曲間で配置` と異なり、前半の配置は保たれます。

---

## SRT合成

SRT合成では、曲ごとのSRTファイルを、アルバム全体のタイムラインへ合わせて1本にまとめます。

基本ルールは次の通りです。

- SRTファイルは選択直後にファイル名昇順で並ぶ
- 曲リストの現在順とSRTリストの現在順を1対1で対応させる
- 各SRTのタイムスタンプに、対応曲の音源00:00位置を加算する
- 字幕番号は1から振り直す
- `album_subtitles.srt` として出力する

曲数とSRTファイル数が一致しない場合は出力できません。

SRT合成は、音源ごとのSRTが「その音源ファイルの00:00」を基準に作られていることを前提にしています。音源ファイルの途中から字幕を作った場合や、別編集後の音源に対するSRTを使う場合は、タイミングが合わない可能性があります。

---

## offset_image.txt

`offset_image.txt` は、各音源の00:00位置を一覧化したファイルです。

例:

```text
01	00:00,000
02	03:50,357
03	07:14,645
```

これは、既存の字幕結合バッチや外部ツールで、曲別SRTをアルバム全体時刻へ変換するための基準として使えます。

このアプリ内の `album_subtitles.srt` 出力を使う場合、通常は別途 `offset_image.txt` を使う必要はありません。ただし、外部ワークフローとの互換性や検証用には有用です。

---

## timesheet.txt

`timesheet.txt` は、動画編集用のタイムシートです。

例:

```text
01	00:00,000	雨あがりの合図
02	03:48,250	次の曲名
03	07:21,430	さらに次の曲名
```

1曲目は常に `00:00,000` です。2曲目以降は、前曲の実音終了と次曲の実音開始の中央時刻です。

この時刻は、曲の開始そのものではなく、**曲間の中央**です。動画編集で画像やタイトルカードを切り替える場合、前曲が終わった瞬間に急に画像を変えるより、曲間中央で切り替えた方が自然なことがあります。このアプリのtimesheetは、その運用を想定しています。

3列目には、音源ファイル名から拡張子を除いた仮タイトルが入ります。正式タイトルではなく、動画編集時の目印として使う想定です。

---

## tracklist.txt

`tracklist.txt` は、YouTubeの説明欄などにそのまま貼るための曲リスト元テキストです。

例:

```text
【Track List】
00:00 01点滅する夜明けのサイン feat. KIRARI_leveled
04:05 02色づく魔法と知らない匂い feat. KIRARI_leveled
08:05 03色褪せないまま踊り出す feat. KIRARI ver.2_leveled
```

1行目は固定で `【Track List】` です。2行目以降に、`MM:SS 仮タイトル` の形式で曲リストが入ります。

時刻は `timesheet.txt` と同じ切替時刻に基づきます。2曲目以降は、前曲の実音終了と次曲の実音開始の中央時刻です。ただし、`tracklist.txt` ではミリ秒を表示せず、3捨4入で秒単位へ丸めます。399msまでは切り捨て、400ms以上は切り上げです。

YouTubeのチャプターとして使う場合は、YouTube側の仕様や説明欄全体の内容により、チャプターとして認識されるかが変わることがあります。このアプリは、チャプター用に使いやすい元テキストを出力するところまでを担当します。

---

## project.json

`project.json` は、作業ファイルです。

この作業自体は長時間かかる処理ではありませんが、次のようなケースでは保存しておく価値があります。

- 音源を一部差し替えて、同じ曲順と曲間で作り直したい
- SRT対応を保持したい
- 実音開始・実音終了の手動調整を残したい
- 後日、アルバムWAVだけ再書き出ししたい
- 動画用timesheetや公開用tracklistだけ再出力したい

### 保存されるもの

project.jsonには、曲順、ファイル名、実音開始・実音終了、音源00:00位置、標準曲間、無音判定設定、処理モード、出力形式、SRT対応情報が保存されます。

### 保存されないもの

音声ファイル本体とSRTファイル本体は保存されません。

これはブラウザアプリの制約であり、設計上の仕様です。project.jsonは、元ファイルを指し示す作業メモであり、元ファイルを埋め込んだパッケージではありません。

### 必要ファイル一覧ダイアログ

音源やSRTを読み込まずにproject.jsonを読み込むと、必要な音声ファイルとSRTファイルの一覧が表示されます。

`一覧をコピー` ボタンで、必要ファイル一覧をクリップボードへコピーできます。作業フォルダ内でファイルを探すときに使えます。

### ファイル差し替え時の考え方

project.json内のファイル名と、現在選択したファイル名が一致する場合は、ファイル名に基づいて並べ替えと設定適用を行います。

ファイル名が一致しないが曲数が同じ場合は、現在の曲順に対してproject.jsonの設定を適用するか確認します。

この動作により、たとえば `track02_old.wav` を `track02_new.wav` に差し替えた場合でも、曲数と順番を揃えれば、以前のタイムライン設定を再利用できます。

ただし、差し替え後の音源で曲頭・曲末の無音量が大きく変わっている場合は、実音開始・実音終了を再確認してください。

---

## 書き出し

### album.wav を出力

現在のタイムラインに基づいてアルバムWAVを書き出します。

実音区間同士が重なっている場合、WAV出力は停止します。このアプリはミキサーではないため、複数音源の同時再生やクロスフェードは行いません。

### album_subtitles.srt を出力

SRTリストと曲リストを1対1対応させて、アルバム全体のSRTを出力します。

### timesheet.txt を出力

曲ごとの画像差し替え目安時刻と仮タイトルを出力します。

### tracklist.txt を出力

YouTube説明欄などに置く曲リスト用テキストを出力します。`timesheet.txt` と同じ時刻を元に、`MM:SS` 形式へ3捨4入して丸めます。

### project.json を読み込み

保存済みの作業状態を読み込みます。音声ファイルやSRTファイル本体は含まれないため、必要ファイルを別途選択します。

### project.json を出力

現在の作業状態を保存します。

### offset_image.txt を出力

各音源の00:00位置を出力します。

## 技術的なTips

### 1. 「曲間」はファイル同士の隙間ではなく実音同士の隙間

標準曲間は、前曲ファイルの末尾と次曲ファイルの先頭の間ではありません。前曲の実音終了と次曲の実音開始の間です。

この違いを理解しておくと、波形表示やoffsetの意味が分かりやすくなります。

### 2. 音源00:00位置はSRT合成の基準

SRTは通常、各曲の音源00:00を基準に作られています。そのため、アルバムSRTを作るには、各音源の00:00がアルバム上のどこに置かれたかが重要です。

この値が曲リストの「音源 00:00位置」であり、`offset_image.txt` にも出力されます。

### 3. timesheetの時刻は曲頭ではなく曲間中央

`timesheet.txt` の2曲目以降の時刻は、次曲の実音開始そのものではありません。前曲実音終了と次曲実音開始の中央です。

動画編集で画像を切り替える場合は、この仕様の方が自然なことがあります。曲開始ぴったりで画像を切り替えたい場合は、`offset_image.txt` や曲リスト上の実音開始時刻を参照してください。

### 4. tracklistの時刻は公開用に丸める

`tracklist.txt` は人間が見る説明欄用なので、ミリ秒は出力しません。`timesheet.txt` と同じ基準時刻を、399msまでは切り捨て、400ms以上は切り上げて `MM:SS` 形式にします。

このため、厳密な編集点は `timesheet.txt`、公開文やYouTube説明欄は `tracklist.txt`、という使い分けができます。

### 5. MP3入力は便利だが、厳密同期にはWAVが安全

MP3はブラウザでデコードされます。MP3にはエンコーダ遅延やパディングがあり、元WAVと完全に同じサンプル位置にならないことがあります。

一般的なアルバムWAV作成や動画用BGMでは問題になりにくいですが、既存SRTや映像と厳密に合わせる場合は、WAV素材を使う方が安全です。

### 6. フェードアウト処理はしない

このアプリはフェードアウトを追加しません。実音終了以降をゼロにするだけです。

フェードアウトを作りたい場合は、事前にDAWや別ツールで音源自体を処理してから、このアプリへ読み込んでください。

### 7. クリックノイズ除去には末尾編集が使える

Sunoなどの生成音源では、曲末尾に小さなクリックノイズや破綻音が入ることがあります。

そのような場合、全体波形右端の `END` を選び、最終曲実音終了をクリックノイズの手前へ調整します。途中曲の場合は、次曲を選択して前曲実音終了を調整します。

### 8. project.jsonはバックアップにもなる

最終的に `album.wav` を出した後でも、project.jsonを一緒に保存しておくと、後日SRTだけ直したい、曲を1つ差し替えたい、timesheetやtracklistだけ再出力したい、という場合に役立ちます。

## よくある使い方

### Sunoなどの生成曲をアルバム化する

1. 各曲をWAVまたはMP3で用意します。
2. ファイル名の先頭に `01_`、`02_` のような番号を付けます。
3. まとめて選択します。
4. 自動配置します。
5. 曲間を確認して `album.wav` を出力します。
6. 必要なら `timesheet.txt` を動画編集用に出力します。

### 字幕付きアルバム動画を作る

1. 曲ごとのSRTを用意します。
2. 音声タイムラインを作ります。
3. SRTファイルを選択し、曲順と対応させます。
4. `album_subtitles.srt` を出力します。
5. 動画作成ツールや編集ソフトで、`album.wav` と `album_subtitles.srt` を使います。

### 一部音源を差し替える

1. 以前の `project.json` を読み込みます。
2. 必要ファイル一覧を確認します。
3. 差し替え後の音源を含めて、同じ曲数の音声ファイルを選択します。
4. ファイル名が変わっている場合は、現在の曲順に適用するか確認します。
5. 波形を確認し、必要なら実音開始・終了を微調整します。
6. 書き出します。

## 注意点と制限

- 出力はWAVのみです。
- MP3出力は行いません。
- フェード、音量補正、クロスフェード、マスタリング処理は行いません。
- 実音区間同士の重なりを含むWAV出力はできません。
- 標準WAVモードは、全WAVが同一形式の場合のみ使われます。
- MP3入力や形式混在入力では、デコードモードになります。
- MP3デコードはブラウザ実装に依存します。
- デコードモードのサンプルレートは、ブラウザでデコードされた後の値を基準にします。
- デコードモードのサンプルレート変換は簡易的な線形補間です。
- 16bit / 24bit / 32bit PCMへ書き出す際、ディザは行いません。
- 通常RIFF/WAVの制約上、4GBを超えるWAVは出力できません。
- project.jsonには音声ファイル本体やSRTファイル本体は含まれません。
- ローカルHTMLアプリのため、保存済みproject.jsonから元ファイルを自動で開くことはできません。
